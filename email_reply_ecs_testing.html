I wanted to share the results from our test deployment of CrowdStrike Falcon components on Amazon ECS with the EC2 launch type. Both the Falcon Sensor for Linux and the Falcon Image Assessment at Runtime (IAR) agent were successfully deployed and verified. A detailed write-up is attached for your reference.

What Was Tested

Component Deployment Method Status
Falcon Sensor for Linux Host-level agent installed via EC2 Launch Template user data Confirmed Running
Falcon IAR (Image Assessment at Runtime) ECS daemon service — Docker socket mode, one container per EC2 instance Confirmed Running
Test Workload (nginx) ECS replica service — used to verify IAR detects newly launched images Scanned by IAR

Verification Results

  • Falcon Sensor — both EC2 instances registered with the Falcon platform with active Agent IDs (AIDs), confirming full host-level protection.
  • IAR — both daemon tasks are running with a 4 GB memory limit and confirmed on task definition revision 4. IAR detected the nginx test workload image via the Docker socket and submitted it for assessment. Results are visible in the Falcon Console under Cloud Security → Image Assessment → Runtime.
